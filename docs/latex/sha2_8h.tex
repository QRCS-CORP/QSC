\begin{DoxyParagraph}{}
The SHA2 and HMAC implementations use two different forms of api\+: short-\/form and long-\/form. ~\newline
The short-\/form api, which initializes the state, processes a message, and finalizes by producing output, all in a single function call, for example; \doxylink{sha2_8h_a5602305c5a0d7e022c82dfafa25e287f}{qsc\+\_\+sha512\+\_\+compute(uint8\+\_\+t\texorpdfstring{$\ast$}{*} output, const uint8\+\_\+t\texorpdfstring{$\ast$}{*} message, size\+\_\+t msglen)}, the entire message array is processed and the hash code is written to the output array. ~\newline
The long-\/form api uses an initialization call to prepare the state, a update call to process the message, and the finalize call, which finalizes the state and generates a hash or mac-\/code. ~\newline
The HKDF key derivation functions HKDF(HMAC(SHA2-\/256/512)), use only the short-\/form api, single-\/call functions, to generate pseudo-\/random to an output array. ~\newline
Each of the function families (SHA2, HMAC, HKDF), have a corresponding set of reference constants associated with that member, example; QSC\+\_\+\+HKDF\+\_\+256\+\_\+\+KEY\+\_\+\+SIZE is the minimum expected HKDF-\/256 key size in bytes, QSC\+\_\+\+HMAC\+\_\+512\+\_\+\+MAC\+\_\+\+SIZE is the minimum size of the HMAC-\/512 output mac-\/code output array.
\end{DoxyParagraph}
